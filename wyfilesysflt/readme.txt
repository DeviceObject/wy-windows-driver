这里谈一谈上下文，包含了实例上下文、文件上下文、流上下文、流句柄上下文。
个各上下文都是跟相应的内核对象匹配的，用于描述该对象的属性的。我们可以自己为不同类型的对象定义不同类型的上下文。

今天在网上查看了相关的资料之后对上下文有了一个具体的认识：
Context －上下文，其实就是附着在某个对象上的一段数据；
Stream Context - 流上下文，也就是FCB（File Control Block）的上下文，文件和FCB是一对一的关系；
Stream Handle Context － 流句柄上下文，也就是FO（File Object）的上下文，一个文件可以对应多个FO，属一对多关系；
Instance Context - 实例上下文，也就是过滤驱动在文件系统的设备堆栈上创建的一个过滤器实例；
Volume Context － 卷上下文，卷就是C，D，E盘以及网络重定向器，一般情况下一个卷对应一个过滤器实例对象，在实际应用上经常用Instance Context来代替Volume Context。

相关的说明里面并没有提及文件上下文，可能它只是作为一个辅助的数据结构而存在吧。
在文件过滤驱动中更多的是看到流上下文和流句柄上下文在发挥作用

原来文件上下文在Vista之后的版本才出现。所以对文件句柄的描述会相对较少。估计普遍是使用流句柄上下文来替代文件上下文的。
在实际应用中一般用流上下文替代文件上下文。
我在启动驱动时遇到的蓝屏就是跟这个有关，在XP环境下去掉文件上下文部分的东西就没问题了。具体真的是不是我说的这样还需要在Win7的环境下试试。
事实证明，上面一行的文字描述错误的。引起蓝屏的问题是创建文件上下文的时候我没有将创建的上下文传出，导致释放上下文的时候读写内存错误！
